### Порождающий шаблон проектирования — строитель (Builder): C#

**Класс `Pizza`** — тип данных для целевых объектов с полями (из вспомогательных классов для них) — строковыми: размером (`size`) и типом корочки (`crust`), а равно логическими: колбасой (`sausage`), сыром (`cheese`), грибами (`mushrooms`) и оливками (`olives`); имеет перегруженный метод `ToString()` для вывода на форму.

**Абстрактный класс `PizzaBuilder`** — рассматриваемый паттерн, обрабатывающий объект `pizza` (уже поле) методами для параметров (`Size(string)` и `Crust(string)`; `Sausage(bool)`, `Cheese(bool)`, `Mushrooms(bool)` и `Olives(bool)`) и сборки с ними (`BuildPizza()`).

**Класс `ThePizzaBuilder`** — определённая реализация `PizzaBuilder` по наследованию, содержащая методы для конфигурирования `pizza` (`Size(string)`, `Crust(string)`, `Sausage(bool)`, `Cheese(bool)`, `Mushrooms(bool)`, `Olives(bool)`): каждый из них возвращает текущий объект, образуя цепочку вызовов.

**Класс `Baker`** — руководитель, использующий объект `pizzaBuilder` для контроля построения в `Bake(…)` с итоговым возвращением.

**Класс `Form1`** (Windows GUI) — основа с виджетами для выбора параметров: два `comboBox` (размер пиццы и тип корочки, устанавливаемые в `Form1_Load(…)` — при запуске) и четыре `checkBox` (булевы); кнопки (обработчики событий) `button1`, создающую объект напрямую (через конструктор), и `button2` — с применением шаблона (с помощью классов `ThePizzaBuilder` и `Baker`); также `richTextBox1`, отображающий результаты.

**Вывод**: паттерн-строитель пошагово конструирует составные объекты (со многими параметрами) другого класса ради удобства их настройки, вынося сложную логику и предоставляя упрощённый интерфейс; однако сам тяжеловесен.
